# 总体架构

## 一、架构分层

```text
Android App
├─ UI 表现层
├─ Workbench 调试工作台层
├─ Orchestrator 能力编排层
├─ Connectors 连接器层
├─ Device Profiles 设备画像层
└─ Core 协议与工具内核层
```

## 二、职责划分

### UI 表现层
负责统一展示设备、终端、配置、日志、JT808、视频、诊断等界面。

### Workbench 调试工作台层
把不同协议进入后的能力，统一收敛成一致的操作体验。

### Orchestrator 能力编排层
根据设备能力、连接器可用性、设备画像规则，决定用什么方式进入设备，并调度动作执行。

### Connectors 连接器层
负责不同接入协议：

- ADB
- Serial
- RNDIS
- SSH
- HTTP
- Raw USB

### Device Profiles 设备画像层
负责描述某类设备的：

- 配置文件位置
- 日志位置
- 数据库位置
- 关键参数提取规则
- 健康检查规则
- 推荐连接方式
- 快捷动作

### Core 协议与工具内核层
负责基础通用能力：

- USB 枚举
- 网络探测
- 命令执行抽象
- 文件操作抽象
- 终端抽象
- ini / sqlite / log / jt808 解析

## 三、架构目标

- 支持多设备，而不是单设备定制
- 后续新增设备时，以补画像和连接器为主
- 后续新增视频、JT808、AI 配置时，不破坏主架构
- 让设备接入、调试、导出诊断形成可复用流程
