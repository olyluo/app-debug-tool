# 核心接口草案：终端与文件抽象

## 一、目标

无论底层是 ADB、串口、SSH 还是其它方式，上层都应该看到统一的终端与文件能力。

## 二、TerminalSession 建议

```kotlin
interface TerminalSession {
    val sessionId: String
    val title: String
    val mode: TerminalMode

    val output: Flow<TerminalChunk>

    suspend fun send(input: String)
    suspend fun resize(columns: Int, rows: Int)
    suspend fun close()
}
```

## 三、文件抽象建议

```kotlin
interface RemoteFileSystem {
    suspend fun list(path: String): List<RemoteFile>
    suspend fun read(path: String): ByteArray
    suspend fun write(path: String, data: ByteArray): WriteResult
    suspend fun delete(path: String): DeleteResult
}
```

## 四、RemoteFile 建议

```kotlin
data class RemoteFile(
    val path: String,
    val name: String,
    val isDirectory: Boolean,
    val size: Long?,
    val modifiedAt: Long?
)
```

## 五、设计原则

- 终端输出使用流模型，适合日志与交互式 shell
- 文件抽象要兼容只读设备和只写受限设备
- 串口终端与 shell 终端可共用大部分 UI，但底层实现不同
