# 首期开发顺序建议

## 一、目标

明确首期开发实施顺序，降低并行开发时的互相阻塞。

## 二、推荐顺序

### 第 1 步：工程骨架
- 创建多模块工程
- 打通 app 启动
- 建立基础导航
- 建立依赖注入入口

### 第 2 步：USB 识别
- 枚举 USB 设备
- 展示 VID/PID
- 展示接口类型
- 判断 ADB / RNDIS / Serial 候选能力

### 第 3 步：ADB Connector
- 探测 ADB 设备
- 打开基础 shell
- 输出终端内容

### 第 4 步：统一终端页
- 展示终端输出
- 输入命令
- 保存会话状态

### 第 5 步：RNDIS 与网络探测
- 探测 IP
- 扫描常见端口
- 输出连接建议

### 第 6 步：Serial Connector
- 打开串口
- 收发文本
- 支持 HEX 查看

### 第 7 步：Profile 与配置中心
- 实现 Generic Linux Profile
- 实现 JT808 AI Recorder Profile
- 读取终端 ID / 车牌 / 服务器 / 白名单

### 第 8 步：日志中心与诊断中心
- 读取关键日志
- 汇总诊断结果
- 导出诊断包

## 三、原则

- 先把可连接打通，再做可理解
- 先做主链路 ADB，再补 RNDIS 和串口
- 先做只读能力，再逐步增加写能力
