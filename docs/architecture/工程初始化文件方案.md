# 工程初始化文件方案

## 一、目标

为 Android 多模块工程提供统一初始化方案，确保后续模块扩展时结构稳定。

## 二、建议根目录文件

```text
settings.gradle.kts
build.gradle.kts
gradle.properties
.gitignore
README.md
PROJECT_PLAN.md
```

## 三、settings.gradle.kts 建议职责

- 定义仓库源
- 定义根项目名
- include 所有首期模块

建议首期 include：

```text
:app
:core:common
:core:usb
:core:network
:core:shell
:core:file
:core:terminal
:core:parser
:core:device
:connectors:connector-adb
:connectors:connector-serial
:connectors:connector-rndis
:profiles:profile-generic-linux
:profiles:profile-jt808-ai-recorder
:features:feature-device-discovery
:features:feature-terminal
:features:feature-config-center
:features:feature-log-center
:features:feature-diagnostics
```

## 四、根 build.gradle.kts 建议职责

- 统一插件版本
- 统一 Kotlin 版本
- 统一 Android 编译参数
- 提供公共依赖约束

## 五、gradle.properties 建议内容

- JVM 参数
- AndroidX 开关
- Kotlin code style
- 并行构建开关

## 六、初始化原则

- 先把模块边界定清楚，再开始写业务代码
- 先搭骨架，再补实现
- 首期不要过早引入过多第三方依赖
