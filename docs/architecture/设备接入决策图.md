# 设备接入决策图

## 一、目标

把“设备插上后先判断什么、先走哪条接入链路”固定成团队统一流程。

## 二、接入决策顺序

```text
设备接入 USB
 -> 枚举 USB 信息
 -> 判断是否支持 ADB
    -> 是：优先 ADB
    -> 否：继续
 -> 判断是否出现 RNDIS / USB 网卡
    -> 是：探测 IP 和常见端口
    -> 否：继续
 -> 判断是否为串口设备
    -> 是：进入串口终端
    -> 否：继续
 -> 判断是否存在 HTTP / 私有接口
    -> 是：进入对应 Connector
    -> 否：进入 Raw USB 分析模式
```

## 三、为什么这样排序

### 1. ADB 优先
因为 ADB 最适合 Linux 设备调试，能力最完整：

- shell
- 文件读取
- 日志查看
- 配置导出

### 2. RNDIS 第二
很多设备会出 USB 网卡，通过网络可以快速探测：

- SSH
- Telnet
- HTTP
- 私有 TCP 服务

### 3. 串口第三
当系统启动不完整、ADB 不可用时，串口常是最可靠兜底。

### 4. Raw USB 最后
Raw USB 用于没有公开协议、需要逆向分析的场景。

## 四、团队统一原则

- 优先选最强、最稳定、最易读的连接方式
- 先做只读探测，再做写操作
- 先判断设备能力，再决定 UI 展示入口
