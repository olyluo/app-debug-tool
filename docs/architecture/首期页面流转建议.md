# 首期页面流转建议

## 一、目标

明确 MVP 阶段页面结构，避免 UI 和底层能力脱节。

## 二、建议页面流转

```text
首页
 -> 设备发现页
 -> 设备详情页
 -> 选择连接方式
 -> 统一工作台
    ├─ 终端
    ├─ 配置
    ├─ 日志
    └─ 诊断
```

## 三、页面说明

### 1. 首页
职责：

- 进入设备发现
- 查看最近设备
- 查看最近诊断结果

### 2. 设备发现页
职责：

- 枚举 USB 设备
- 展示识别到的能力
- 给出推荐连接方式

### 3. 设备详情页
职责：

- 展示 fingerprint
- 展示识别到的 connector
- 展示匹配到的 profile
- 展示关键能力

### 4. 统一工作台
首期建议 4 个页签：

- 终端
- 配置
- 日志
- 诊断

## 四、后期扩展

后续可增加：

- JT808
- 视频
- 文件
- 动作脚本
