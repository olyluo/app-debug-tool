# 动作脚本规范

## 一、目标

把常见调试行为沉淀成结构化动作，供不同设备重复使用。

## 二、动作示例

- 读取设备基础信息
- 导出 product.ini
- 导出 JT808 配置
- 检查服务器参数
- 检查白名单
- 导出日志片段
- 重启业务服务
- 开启 Wi-Fi 热点
- 导出诊断包

## 三、动作模型建议

- id
- title
- description
- requiredCapabilities
- executionPlan
- expectedOutputs
- rollbackHint

## 四、原则

- 优先读操作
- 高风险动作要明确提示
- 输出结构化结果，便于 UI 呈现
