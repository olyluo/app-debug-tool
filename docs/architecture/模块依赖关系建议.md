# 模块依赖关系建议

## 一、目标

模块依赖必须保持单向、清晰、可维护，避免后期出现循环依赖和协议耦合。

## 二、推荐依赖方向

```text
app
 -> features
 -> profiles
 -> connectors
 -> core

features
 -> profiles
 -> connectors
 -> core

profiles
 -> core

connectors
 -> core

core
 -> 不依赖上层
```

## 三、详细说明

### app
只做装配和导航，不承载具体协议实现。

### features
面向用户任务，依赖抽象能力，不直接写底层协议细节。

### profiles
负责设备理解，不负责连接建立。

### connectors
负责连接建立和会话暴露，不负责页面组织。

### core
是最底层平台能力层，必须保持稳定。

## 四、依赖控制原则

- core 不能依赖 connectors / profiles / features / app
- connectors 不依赖 features
- profiles 不依赖 features
- feature 尽量依赖接口而不是依赖具体实现
- app 只做最终组装
