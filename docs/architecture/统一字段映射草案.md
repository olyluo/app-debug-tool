# 统一字段映射草案

## 一、目标

不同设备虽然配置文件结构不同，但业务上关心的字段高度重复。

本草案用于定义：

- UI 层统一展示什么字段
- Profile 层如何把不同设备的原始字段映射成统一字段
- 后续新增设备时优先补哪些映射

## 二、总原则

- UI 只面向统一字段，不直接面向原始 ini 键名
- Profile 负责把设备原始字段映射成统一字段
- 同一个统一字段允许多个候选来源
- 统一字段名保持英文、语义明确、长期稳定

## 四、映射优先级建议

### 文本字段
优先顺序：

1. 运行中的主配置
2. profile 指定的主数据库字段
3. 备份配置
4. 推断值

### 布尔/能力字段
优先顺序：

1. 运行态探测
2. 主配置字段
3. profile 推断

## 五、示例

### 示例：车牌号

统一字段：`licensePlateNo`

可能来源：

- `/userdata/product.ini` 的 `licenseplateno`
- 数据库中的车辆基础表
- 平台参数缓存表

### 示例：第一主服务器

统一字段：`mainServerAddr1`

可能来源：

- `/userdata/product.ini` 的 `mainserveaddr1`
- 平台参数库映射
- 远程参数区 `tparammainserveraddr`

### 示例：白名单

统一字段：`avWhitelist`

可能来源：

- `avwhitelist`
- 运行态 JT808 白名单接口返回

## 六、后续落地建议

- 在 `core:device` 中定义统一字段模型
- 在 `profiles` 中实现原始字段到统一字段的映射器
- 在 `feature-config-center` 中只依赖统一字段模型
