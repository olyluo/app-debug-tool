# 模块划分说明

## 一、模块设计目标

项目采用多模块结构，不按单一设备拆分，而按平台能力拆分。

目标：

- 便于后续支持更多 Linux + AI + JT808 设备
- 降低不同协议之间的耦合
- 让连接方式、设备画像、功能页面可以独立演进
- 为后期视频接入、私有协议接入预留空间

## 二、一级模块划分

```text
app/
core/
connectors/
profiles/
features/
docs/
```

### app
Android 主入口模块，负责：

- Application 初始化
- 导航
- 主题
- 全局依赖注入装配
- 首页与工作台宿主

### core
平台基础内核，负责：

- USB 枚举
- 网络探测
- Shell 抽象
- 文件抽象
- 终端抽象
- 配置/日志/数据库解析
- 设备能力与设备指纹模型

### connectors
连接器插件模块，负责不同接入方式：

- ADB
- 串口
- RNDIS
- SSH
- HTTP
- Raw USB

### profiles
设备画像模块，负责：

- 匹配某类设备
- 提供配置路径
- 提供日志路径
- 提供关键参数提取规则
- 提供健康检查规则
- 提供快捷动作

### features
功能模块，负责最终用户使用的页面和功能：

- 设备发现
- 终端
- 文件浏览
- 配置中心
- 日志中心
- JT808 中心
- 诊断中心
- 脚本中心
- 视频中心

### docs
项目文档与规范。

## 三、设计原则

- 上层依赖抽象，不依赖具体连接器实现
- profile 不直接操作 UI
- connector 不直接写业务页面逻辑
- feature 不直接耦合某一种设备型号
