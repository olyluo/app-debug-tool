# 新同事接手阅读顺序

## 目标

让第一次接手这个项目的人，不需要问太多问题，只看文档就能知道项目为什么这么设计、现在做到哪里、接下来先做什么。

## 第一轮：先看懂项目

按顺序阅读：

1. `README.md`
2. `START_HERE.md`
3. `docs/architecture/5分钟看懂项目.md`
4. `docs/architecture/总体架构.md`

读完后应该知道：

- 项目目标是什么
- 这个项目面向什么类型设备
- 为什么采用 Connector + Profile + Feature 结构

## 第二轮：看懂模块

按顺序阅读：

1. `docs/architecture/模块划分说明.md`
2. `docs/architecture/core模块说明.md`
3. `docs/architecture/connectors模块说明.md`
4. `docs/architecture/profiles模块说明.md`
5. `docs/architecture/features模块说明.md`

读完后应该知道：

- 模块怎么拆
- 每层负责什么
- 为什么不能随便跨层依赖

## 第三轮：看懂首期怎么落地

按顺序阅读：

1. `docs/architecture/首期模块清单.md`
2. `docs/architecture/首期里程碑.md`
3. `docs/architecture/首期开发顺序建议.md`
4. `docs/architecture/首期页面流转建议.md`
5. `docs/architecture/开工执行清单.md`

读完后应该知道：

- 首期做哪些模块
- 先做什么，后做什么
- 页面怎么走

## 第四轮：看懂代码抽象

按顺序阅读：

1. `docs/architecture/核心接口草案-Connector.md`
2. `docs/architecture/核心接口草案-DeviceProfile.md`
3. `docs/architecture/核心接口草案-Capability与Device.md`
4. `docs/architecture/核心接口草案-终端与文件抽象.md`

读完后应该知道：

- 连接器接口怎么设计
- 设备画像接口怎么设计
- Device / Capability 怎么建模
- 终端和文件抽象怎么统一

## 第五轮：看懂设备语义

按顺序阅读：

1. `docs/profiles/通用Linux设备画像.md`
2. `docs/profiles/JT808设备画像.md`
3. `docs/profiles/Rockchip记录仪画像.md`
4. `docs/architecture/配置中心字段规划.md`

读完后应该知道：

- 设备画像如何落到真实设备
- 哪些字段最关键
- 后续新增设备要补什么
